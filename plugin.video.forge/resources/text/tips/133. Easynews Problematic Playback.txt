
Easynews is a great source, but there have always been issues with playback of found sources. Often the playback fails on the first try, and you need to try the same source again for the file to play.

There is a setting in Forge to help mitigate this, and you can choose the method that best suits you.

The setting is found in Settings->Scrapers->Easynews. Setting name is "Playback Method".

Below is a brief explanation of the options you have here....

NONE: No special method used. Forge will attempt to resolve the Easynews link just as it would any other.
RETRY: Forge retries the Easynews link if it fails the first time. This is slow and doesn't work on some hardware.
NO SEEK: Forge tells Kodi the file is not seekable, which makes resolving Easynews links much quicker and more reliable. The file will usually play first try and quickly. The downside is you will not be able to seek forward/backward outside of the current cached data.
BOTH: Forge will use NO SEEK, and will also allow for a RETRY. This is probably not needed for most setups.
